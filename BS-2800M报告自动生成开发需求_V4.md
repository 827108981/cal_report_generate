# BS-2800M 校准报告自动生成开发需求 V4

## 1. 最终确认规则

| 项目 | 规则 |
|---|---|
| 首页序列号 | M1/M2 序列号合并，去掉末尾 `M1`、`M2` 后缀 |
| 校准日期和有效期 | 以 Excel 第 5 行为准 |
| CSV | 不使用 `1111.csv`、`SampleResult_20250113_142443.csv` |
| 图片 | 当前阶段不读取图片文件夹 |
| 模块 | 支持单 M1、单 M2、M1+M2 |
| 结论 | Pass 类统一显示为 `■ Pass` |
| 公共章节 | M1/M2 合并，规则为“谁有有效数据就取谁” |

## 2. 公共章节最终规则

公共章节指最终报告中只出现一套，但数据可能分布在 M1/M2 两份 Excel 中的章节：

- 常规生化项目精密度。
- 常规生化项目正确度。
- 电解质正确度。
- 电解质精密度。
- 电解质线性范围。
- 电解质稳定性。
- 电解质交叉污染。

最终规则：

1. 按项目维度合并 M1/M2 数据。
2. 哪个模块有有效数据，就取哪个模块。
3. M1 有效、M2 无效：取 M1。
4. M2 有效、M1 无效：取 M2。
5. M1/M2 都无效：报错，提示该项目缺少数据。
6. M1/M2 都有效：记录冲突并报错，要求人工确认，不静默选择。

有效数据定义：

- 单元格不为空。
- 单元格不是 `/`。
- 单元格不是 Excel 错误值，例如 `#DIV/0!`、`#VALUE!`、`#REF!`、`#N/A`。
- 对于整列项目，至少有关键测量值、计算值和结论能组成完整结果。

## 3. 输入文件

程序输入：

- Word 模板：`BS-2800M系列_全自动生化分析仪仪器校准报告_V1.0_CH.docx`
- M1 Excel：可选
- M2 Excel：可选
- 配置文件：建议 `report_config.yaml`
- 输出 Word 路径

当前阶段不读取：

- CSV 文件。
- `M1/`、`M2/` 图片文件夹。

## 4. 配置文件

建议配置：

```yaml
hospital_name: 铜川矿务局中心医院
department: 检验科
manufacturer: 深圳迈瑞生物医疗电子股份有限公司
calibrator: 孙彤彤

modules:
  - M1
  - M2

common_merge_mode: valid-data
```

字段说明：

| 字段 | 说明 |
|---|---|
| `hospital_name` | 医院名称 |
| `department` | 使用科室 |
| `manufacturer` | 仪器制造商 |
| `calibrator` | 校准人员 |
| `modules` | 生成模块，支持 `[M1]`、`[M2]`、`[M1, M2]` |
| `common_merge_mode` | 公共章节合并模式，固定为 `valid-data` |

不需要配置：

- `serial_number`：从 Excel 第 3 行读取并合并。
- `calibration_date`：从 Excel 第 5 行读取。
- `valid_until`：从 Excel 第 5 行读取。

## 9. 日期和数值格式

日期从 Excel 第 5 行读取，建议格式：

```text
2024年11月4日
```

数值格式：

- 吸光度按目标报告格式，通常保留 4 位小数并去除多余尾零。
- 百分比显示为 `0.28%`、`1.50%`、`≤2%` 等。
- 温度显示 `℃`。
- 结论统一 `■ Pass`。

开发时优先保留 Excel 显示格式；若无法读取显示格式，再用程序格式化。

## 10. 计算要求

为避免 Excel 公式缓存错误，建议程序复算：

- Mean。
- SD。
- CV。
- 最大值、最小值。
- 稳定性。
- 线性回归 R2、R、a、b。
- 加样均值。
- 加样误差。
- 携带污染率。
- 电解质波动百分比。
- 交叉污染 CLH/CHL。
- Pass/Fail。

第一版可以先读取 Excel 已有计算结果；若遇到错误值，必须按公共章节有效数据规则处理，不能直接写入 Word。

## 11. 输出和校验

程序输出：

- 最终 `.docx`。
- `generation_log.json`：记录每个模块章节和公共项目的数据来源。
- `validation_report.md`：记录缺失、冲突、错误值、格式问题。

校验内容：

- 序列号是否合并成功。
- 日期是否来自 Excel。
- 单模块报告是否删除另一模块章节。
- 双模块报告是否包含 M1/M2 两套模块章节。
- 公共章节每个项目实际来自 M1 还是 M2。
- 是否存在未处理的 `#DIV/0!` 等错误值。
- 是否存在 M1/M2 同项目都有效的冲突。
- 是否存在 M1/M2 同项目都无效的缺失。
- 所有通过结论是否统一为 `■ Pass`。

## 12. 开发里程碑

第一阶段：读取 M1/M2 Excel，生成无图片双模块报告。

第二阶段：实现单模块报告。

第三阶段：实现公共章节按有效数据合并，并输出每个项目的数据来源日志。

第四阶段：统一日期、数字、百分比、结论格式。

第五阶段：生成校验报告。

第六阶段：如后续需要，再实现图片自动插入。
